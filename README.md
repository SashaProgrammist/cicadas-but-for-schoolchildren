## ход мероприятия 

### Первый этап: Книжный шифр с шаблонами

#### Цель:
Участники должны разгадать зашифрованное сообщение, используя книги и шаблоны с отверстиями.

#### Инструменты:
1. **Библиотека книг**: Доступ к трем книгам, из которых в начале доступна только одна.
2. **Шаблоны**: Набор шаблонов (листы с прямоугольными отверстиями) для наложения на страницы книг.
3. **Команды бота**:
    - `get <название книги> <номер страницы>`: Возвращает указанную страницу из книги.
    - `impose <номер шаблона>`: Накладывает выбранный шаблон на последнюю полученную страницу.

#### Процесс:

1. **Начало**:
    - Участники получают инструкции о том, как пользоваться командами `get` и `impose`.
    - Доступна первая книга.

2. **Получение страницы книги**:
    - Участники используют команду `get` для получения страницы книги. Пример: `get "Book1" 23` вернет 23-ю страницу первой книги.

3. **Использование шаблонов**:
    - Участники могут накладывать шаблоны на полученные страницы, используя команду `impose`. Пример: `impose A` накладывает первый шаблон на последнюю полученную страницу.
    - Шаблоны имеют отверстия, через которые видны только определенные буквы или символы на странице, что может создавать сообщение или подсказку.

4. **Разгадка шифра**:
    - Участники должны использовать комбинацию страниц книг и шаблонов, чтобы разгадать зашифрованное сообщение.
    - На первом этапе доступна только одна книга и несколько шаблонов. Как только участники разгадывают часть шифра, они могут получить доступ ко второй книге.

5. **Переход к следующему этапу**:
    - Когда участники правильно разгадывают все части шифра, бот предоставляет им код для перехода к следующему этапу или дает доступ к следующей книге и новым шаблонам.

### Пример сценария:

1. **Получение первой страницы**:
    - Команда: `get "Book1" 5`
    - Бот возвращает текст 5-й страницы первой книги.

2. **Наложение шаблона**:
    - Команда: `impose 1`
    - Бот возвращает страницу с наложенным первым шаблоном, показывая участникам определенные буквы или символы.

3. **Интерпретация**:
    - Участники интерпретируют увиденные буквы или символы и пытаются понять часть шифра.

4. **Переход к следующей странице или книге**:
    - Команда: `get "Book1" 12`
    - Бот возвращает текст 12-й страницы первой книги.
    - Команда: `impose 2`
    - Бот возвращает страницу с наложенным вторым шаблоном.

5. **Разгадка**:
    - Участники продолжают процесс, пока не разгадают сообщение.

### Дополнительные идеи:

- **Подсказки**: Добавьте возможность получения подсказок за определенные действия или через команду `hint`.
- **Логирование**: Ведите лог действий участников, чтобы отслеживать их прогресс и в случае необходимости предоставить помощь.
- **Прогресс**: Сообщайте участникам об их прогрессе и количестве оставшихся шагов.

### Пример диалога с ботом:

1. Участник: `get "Book1" 5`
2. Бот: `Текст 5-й страницы книги "Book1".`
3. Участник: `impose 1`
4. Бот: `Текст 5-й страницы книги "Book1" с наложенным первым шаблоном. Видимые буквы: H, E, L, L, O`

Такой подход обеспечит увлекательный и интерактивный первый этап, который задаст тон всей игре и вовлечет участников в процесс разгадки шифров.

### Второй этап: Поиск информации в интернете

Второй этап, предполагающий поиск информации в интернете, может быть очень увлекательным и познавательным. Вот как можно его организовать и обеспечить однозначность ответов:

#### Цель:
Участники должны найти определённую информацию в интернете, которая будет являться частью следующего шифра.

#### Инструменты:
1. **Ключевые слова и подсказки**: Участники получают подсказки и ключевые слова для поиска.
2. **Контрольные точки**: Чёткие и конкретные вопросы, ответы на которые можно однозначно найти в интернете.
3. **Валидация ответов**: Механизм проверки правильности ответов.

#### Процесс:

1. **Начало**:
    - Бот высылает участникам задания и ключевые слова для поиска информации.

2. **Задания**:
    - Участники получают несколько вопросов, ответы на которые можно найти в интернете. Например, это могут быть вопросы, требующие поиска информации на сайтах кафедр университета или на страницах с определёнными научными статьями.

3. **Поиск информации**:
    - Участники используют ключевые слова для поиска ответов на вопросы.

4. **Валидация ответов**:
    - Участники отправляют ответы боту, который проверяет их правильность.
    - Бот может использовать определенные контрольные фразы или шаблоны для проверки правильности ответов.

5. **Переход к следующему этапу**:
    - После того как все ответы будут правильно найдены, бот предоставит следующий ключ или код для перехода к следующему этапу.

### Обеспечение однозначности ответов:

1. **Четко сформулированные вопросы**:
    - Вопросы должны быть составлены таким образом, чтобы на них был только один правильный ответ. Например:
      - "Какое официальное название кафедры ИИТУС?"
      - "Какой номер страницы у раздела 'Абитуриенту' на сайте кафедры ИИТУС?"

2. **Использование контрольных точек**:
    - Вопросы могут быть связаны с конкретными контрольными точками, которые легко проверить. Например:
      - "Посетите сайт кафедры ИИТУС и найдите номер контактного телефона."
      - "Скачайте документ 'Методические указания' с сайта кафедры и найдите номер страницы, на которой описаны правила приёма."

3. **Валидация через бот**:
    - Бот может проверять ответы с использованием заранее заданных контрольных фраз или шаблонов.
    - Например, для проверки номера телефона можно использовать регулярные выражения.

4. **Использование уникальных идентификаторов**:
    - Вопросы могут требовать ответы, которые содержат уникальные идентификаторы, такие как номера страниц, специальные коды или специфическую терминологию.

5. **Предоставление подсказок**:
    - Если участники сталкиваются с трудностями, бот может предоставлять дополнительные подсказки, которые помогут сузить область поиска.

### Пример сценария:

1. **Начало этапа**:
    - Бот: "Ваше задание — найти информацию на сайте кафедры ИИТУС. Вот ваши вопросы:
      1. Какой номер телефона указан на странице 'Контакты'?
      2. В каком году была основана кафедра?
      3. Какое официальное название кафедры?"

2. **Поиск информации**:
    - Участники используют ключевые слова для поиска информации в интернете и на сайте кафедры.

3. **Отправка ответов**:
    - Участники отправляют ответы боту: 
      - `answer 1 <номер телефона>`
      - `answer 2 <год>`
      - `answer 3 <название>`

4. **Валидация ответов**:
    - Бот проверяет ответы на правильность. 
    - Бот: "Ответ на вопрос 1 правильный. Переходите к следующему вопросу."
    - Если ответ неверный: "Ответ на вопрос 2 неверный. Попробуйте еще раз."

5. **Переход к следующему этапу**:
    - После правильного ответа на все вопросы бот отправляет участникам код для следующего этапа.

### Пример диалога с ботом:

1. Участник: `answer 1 +7 (495) 123-45-67`
2. Бот: "Ответ на вопрос 1 правильный."
3. Участник: `answer 2 1975`
4. Бот: "Ответ на вопрос 2 неправильный. Попробуйте ещё раз."
5. Участник: `answer 2 1980`
6. Бот: "Ответ на вопрос 2 правильный."
7. Участник: `answer 3 Кафедра информационных и измерительных технологий и систем управления`
8. Бот: "Ответ на вопрос 3 правильный. Поздравляем, вы прошли второй этап!"

### Подведение итогов:

Такой подход позволит сделать второй этап интересным и обучающим, а также обеспечит однозначность ответов через чётко сформулированные вопросы и автоматическую валидацию ответов ботом.


Связать два этапа можно, сделав так, чтобы разгаданные шифры из первого этапа стали необходимыми подсказками или ключами для выполнения задач второго этапа. Вот как это можно организовать:

### Концепция связи двух этапов

#### Первый этап: Книжный шифр с шаблонами

- Участники разгадывают шифры, используя страницы книг и шаблоны.
- На каждом этапе разгадывания они получают части ключа или подсказки для следующего этапа.

#### Второй этап: Поиск информации в интернете

- Разгаданные части ключа из первого этапа нужны для нахождения правильной информации в интернете.
- Эти ключи помогают ответить на вопросы или сузить область поиска.

### Подробный план

#### Первый этап:

1. **Начало**:
    - Участники получают инструкции и начинают с первой книги и шаблонов.
    - Они должны использовать команды `get` и `impose` для получения текста и наложения шаблонов.

2. **Разгадка шифров**:
    - На каждой странице с наложенным шаблоном видны буквы или символы, составляющие части ключа.
    - Например, на странице 5 книги 1 с шаблоном 1 видны буквы "A", "B", "C".

3. **Сбор ключа**:
    - Участники собирают все части ключа. Например, после нескольких страниц у них получается: "ABCD1234".
    - Этот ключ будет необходим на втором этапе.

4. **Переход ко второму этапу**:
    - Когда все части ключа собраны, бот сообщает, что они переходят ко второму этапу.

#### Второй этап:

1. **Использование ключа**:
    - Бот сообщает, что для поиска информации в интернете участникам понадобится ключ, который они собрали на первом этапе.

2. **Задания и подсказки**:
    - Участники получают задания, для выполнения которых нужен ключ.
    - Пример задания: "Используйте ключ 'ABCD1234' для поиска на сайте кафедры ИИТУС. Найдите страницу с этим кодом и ответьте на следующие вопросы."

3. **Поиск информации**:
    - Участники ищут информацию, используя ключ для нахождения правильных страниц.
    - Например, на сайте кафедры есть скрытая страница с URL, содержащим "ABCD1234", где размещены необходимые данные.

4. **Валидация ответов**:
    - Участники отправляют ответы боту, который проверяет их правильность.
    - Правильные ответы позволяют перейти к следующему этапу или получить дополнительные подсказки.

### Пример сценария:

#### Первый этап:

1. **Получение первой страницы**:
    - Участник: `get "Book1" 5`
    - Бот: `Текст 5-й страницы книги "Book1".`

2. **Наложение шаблона**:
    - Участник: `impose 1`
    - Бот: `Текст с наложенным шаблоном: видимые буквы "A", "B", "C".`

3. **Сбор ключа**:
    - Участник продолжает и собирает ключ: "ABCD1234".

4. **Переход ко второму этапу**:
    - Бот: "Поздравляем, вы собрали ключ 'ABCD1234'. Используйте его для следующего задания."

#### Второй этап:

1. **Получение задания**:
    - Бот: "Для второго этапа используйте ключ 'ABCD1234'. Найдите на сайте кафедры ИИТУС страницу с этим кодом и ответьте на следующие вопросы:
      1. Какой номер телефона указан на этой странице?
      2. В каком году была основана кафедра?"

2. **Поиск информации**:
    - Участники ищут страницу с ключом "ABCD1234" на сайте кафедры ИИТУС.

3. **Отправка ответов**:
    - Участник: `answer 1 +7 (495) 123-45-67`
    - Бот: "Ответ на вопрос 1 правильный."
    - Участник: `answer 2 1975`
    - Бот: "Ответ на вопрос 2 правильный. Поздравляем, вы прошли второй этап!"

### Пример диалога с ботом:

1. Участник: `get "Book1" 5`

3. Участник: `impose 1`

Такой подход не только связывает этапы, но и делает их взаимозависимыми, что добавляет элемент последовательности и логики в игру, усиливая вовлеченность участников.
